前面的章节里介绍了一些常用的动作。相信你也一定掌握了一些动作的基本用法，如把文本、文件等项目发送至动作对它们进行处理。本章将补充并总结 LaunchBar 动作的使用及简单动作的制作方法。

什么是动作

一个动作本身其实就是一个脚本，用于实现一个或一连串功能，如对系统的操作、对文本、文件的处理等。动作是 LaunchBar 最核心的功能之一，大大拓展了 LaunchBar 的功能性，几乎可以覆盖到使用电脑时的各个方面。
在之前的章节中列举了大量的动作实例。想要知道你的 LaunchBar 中总共都有哪些动作，可以在 LaunchBar Index 里查看，右边动作列表的下方能看到所选动作的输入和输出项目，也可以点击 Open in Action Editor 来查看这个动作的具体内容（Action Editor 将在接下来的章节中介绍）。

有输入项目的动作用法总结

对于有输入项目（网址、文本、文件等）的动作，使用方法往往是双向的，既可以把项目发送给此动作进行处理，也可以先选择此动作，再输入项目。

动作和服务的区别

服务是由系统或所安装的软件提供的功能，无法查看其原码，更无法修改。动作是 LaunchBar 内置的或安装的功能， 绝大部分内置动作和所有用户动作都可以来查看其具体内容，用户动作也可以加以修改。
在使用方法上服务和动作没有区别。

制作动作

除了下载安装网上的动作，也可以自制适合自己的动作，来简化日常某些繁琐机械的操作。接下来通过介绍几个简单动作的制作过程，来说明制作动作的方法。读完本节，你可以举一反三，把有趣的想法化为 LaunchBar 中可行的功能。

Action Editor

首先需要介绍的是 LaunchBar 提供的动作编辑器 Action Editor。借助它我们可以轻松制作各种从简单到复杂的动作。在 LaunchBar 中输入缩写如 ACTED 找到它：

左侧边栏是 LaunchBar 中所有可查看及可编辑的动作列表，通过其上面的下拉菜单可以选择所列出的动作类别：
  • All Actions：所有动作，如果点击列表右下方的小锁子图标，可以显示和隐藏可查看的内置动作，但这些内置动作不可直接在 Action Editor 中删除或修改，但可以复制一个副本对副本进行修改；
  • Custom Actions：用户动作，即所有下载的及自制的非内置动作；
  • My Actions：自制动作，即标有自己的动作标识前缀的动作；
  • Recently Modified：最近编辑过的动作。
Action Editor 窗口右边是所选动作的细节内容，分别列在三个标签页上：
  • General：用于管理一般设置；
  • Scripts：用于管理脚本文件；
  • Resources：用于管理资源文件， 如图标等。

你的第一个 LaunchBar 动作

也许你在少数派上读过不少关于 Terminal 命令技巧的文章。当你遇到一些常用的命令，可能需要把它们保存在笔记里，需要用到某一条命令的时候，直接从笔记里复制到 Terminal 里运行它，或发送到 LaunchBar 的 Run Terminal Command 和 Run iTerm Command 动作上运行它。其实这些命令，都可以把它们做成一个简单的 LaunchBar 动作，以便随时调用。
例如：少数派上 @iTumbledSea 的文章《Mac 基础教程：如何让 Finder 显示隐藏文件和文件夹》中介绍了如何隐藏和恢复显示桌面上的文件，用到的命令分别为：
  • 隐藏桌面文件：
    defaults write com.apple.finder CreateDesktop -bool false; killall Finder
  • 恢复显示桌面文件：
    defaults write com.apple.finder CreateDesktop -bool true; killall Finder

下面我们以隐藏桌面文件这个简单的动作为例，介绍一下制作 LaunchBar 动作的具体流程。

1. 新建动作

打开 Action Editor 后，点击左侧边栏左下角的加号图标，选择 New Action，新建一个动作，将其命名为「Hide Everything on Desktop」：
此时我们看到右侧的 General 标签页中，Action Name、Action Identifier 和 File Name 三项自动填充。其中，Action Name 为这个动作的名称，File Name 为这个动作文件的名称，这两个名称可以不同，可以直接在对应的文本框中进行修改。文件名的后缀名为 .lbaction，代表它是一个 LaunchBar 的动作。Action Identifier 中是这个动作的标识，其前缀是你在设置中修改的。

2. 选择脚本语言

接下来我们要创建这个动作的脚本，其内容非常简单，就是隐藏桌面文件的一行命令。运行在 Terminal 中的命令被称为 Shell 命令，因此这个动作的脚本是一个 Shell 脚本。点开 Scripts 标签页，在右上角的下拉菜单中选择要使用的脚本语言 Shell Script：

3. 编辑脚本

点击选择脚本语言的下拉菜单右侧的 Edit 按钮，把这句命令：defaults write com.apple.finder CreateDesktop -bool false; killall Finder 粘贴到弹出的文本编辑器中，保存并退出。
这样我们的动作功能部分就完成了，可以直接在 LaunchBar 上调用它查看使用效果了。

4. 选择图标

由于我们没有为这个动作设置图标，所以它在 LaunchBar 中的图标就是一个简单的齿轮形状。

要为它设置图标，有四种常见的方法：使用 Font Awesome Icons 、使用 Emoji、使用资源文件和调用应用图标。
  1. 使用 Font Awesome Icons：Font Awesome 是一套丰富的图标字体库，LaunchBar 可以直接调用这些图标作为动作图标。在 LaunchBar 中输入缩写如 FONT 找到 Font Awesome Icons，按回车或 → 方向键就可以看到 Font Awesome 中所有的免费图标了：

    再进一步通过二级检索在这些图标中进一步筛选，找到合适的图标。如这个隐藏桌面文件的动作，我们可以选择一个被划掉的眼睛作为图标，输入 EYE 就可以找到它：

    在它上面按 → 方向键，在图标的各项信息中选择第二项 font-awesome:fa-eye-slash，即 Icon Identifier（图标标识）。按 ⌘ C 把这串字符复制下来。
    在 Action Editor 中选择了这个动作之后，选择 General 标签页上的第一项 Action Icon，在其中粘贴这个图标标识，则动作的图标就设置完成了：
  2. 使用 Emoji：也可以直接把 Emoji 输入到 Action Icon 中，就可以直接把它作为动作图标了：

    如何快速输入 Emoji 呢？别忘了在章节《文本操作 | 文本输入 | 快速输入》中的方法。
  3. 使用资源文件：如果在 Font Awesome Icons 和 Emoji 中没有找到合适的图标，也可以把任意图片文件作为动作图标。
    在 Action Editor 中选择了动作之后，选择 Resources 标签页，点击下方的加号选择图片文件加入资源列表，或直接从 Finder 中把图片文件拖入此资源列表。再在 General 标签页中的 Action Icon 填入文件名称，就可以设此图片为动作图标了：
  4. 调用应用图标：直接在 General 标签页中的 Action Icon 中输入应用程序的路径，则这个动作就拥有了和应用相同的图标：

    选择好图标之后，你就已经做好了你的第一个 LaunchBar 动作。
依照以上的步骤，不妨试试把以下几条命令分别做成动作：
  • 重新排列 LaunchPad 图标：
    defaults write com.apple.dock ResetLaunchPad -bool true && killall Dock
  • 在 Finder 中显示隐藏文件和文件夹：
    defaults write com.apple.finder AppleShowAllFiles -boolean true ; killall Finder
  • 在 Finder 中恢复不显示隐藏文件和文件夹：
    defaults write com.apple.finder AppleShowAllFiles -boolean false ; killall Finder
  • 在 Finder 标题栏显示完整路径：
    defaults write com.apple.finder _FXShowPosixPathInTitle -bool YES

查看动作文件

在一个动作上点击右键，选择 Show in Finder，可以打开这个动作所在的文件夹。一般情况下：
  • 内置动作存放路径为：
    /Applications/LaunchBar.app/Contents/Resources/Actions/
  • 用户动作存放路径为：
    /Users/用户名/Library/Application Support/LaunchBar/Actions/
动作文件的后缀为 .lbaction，每个动作文件都是一个包，双击它可以安装它，点击右键选择 Show Package Contents（查看包内容）可以查看它其中的配置、脚本以及资源文件。
在 Finder 中查看内置动作文件时可以对它进行修改或删除，但强烈建议不要这样做，如果你不需要某个内置动作，可以把它在 LaunchBar Index 中禁用掉。如果想要对内置动作进行修改，可以在 Action Editor 中为其创建副本，在副本中进行修改，再原有的内置动作在 LaunchBar Index 中禁用掉。

朗读文字

有时起床洗完澡，当我看到电脑上弹出一篇有趣的文章，我会把全文选中发送给 LaunchBar 的朗读文字动作，把耳机塞到耳朵里，一边吹头发，一边让电脑帮我读文章。

这个 Read Text in Chinese 动作，选择的语言为 AppleScript，脚本内容如下：
on handle_string(_content)
say _content using "Ting-Ting"
display notification "已经朗读完文本的所有内容。" with title "朗读完毕"
end handle_string
  • 第一行表示此动作获取了一个文本，即要朗读的文本，其名称为 _content；
  • 第二行是用 Ting-Ting 这个语言包朗读这个文本；
  • 第三行是在朗读完成之后弹出「朗读完毕」的通知；
  • 第四行是结束对文本的处理。
其中第二行中用的语言包，要预先在电脑上安装，才可以使用它。安装方法如下：
在系统设置中，依次点开 Accessibility → Speech → System Voice → Customize…，在其中选择语言包 Ting-Ting，确定安装。
另外还需要在 Action Editor 的 Scripts 标签页中勾选如下项目：

  • Run in background：此动作在后台运行；
  • Requires argument：选择此动作按回车后可以输入文字；
  • Accepts string argument：此动作接受字符串。
再选择好图标，这个动作就完成了。

Google 翻译

我有时需要把一段文字翻译成中文、英文或者德文，每次打开 Google 翻译再输入文字有点繁琐。后来我制做了这个动作，可以一次性把输入的文本翻译成三种语言：

首先我用 Homebrew 下载了命令行软件 translate-shell。操作方法是，在终端中输入如下命令：
brew install translate-shell
如果你还不了解 Homebrew 是什么，请阅读少数派上 @Minja 的文章《像 Mac 高手一样管理应用，从 Homebrew 开始》。translate-shell 是一个借助 Google 翻译的命令行翻译工具，我需要用到的命令为：
trans -brief :zh 要翻译的内容
trans -brief :en 要翻译的内容
trans -brief :de 要翻译的内容
这三行命令分别用于获取中、英、德文的译文：

如果你想要了解 translate-shell 的更多用法，请参阅其官网。
下面需要把这个命令写成动作脚本。这个动作我选择的语言是 Python，因为可以方便地获取命令行软件返回的文字，且可以简单避免使用 AppleScript 找不到软件路径的问题。
脚本内容如下：
import sys
import json
import subprocess as sp
import os

my_env = os.environ.copy()
my_env["PATH"] = "/usr/local/bin:" + my_env["PATH"]

items = []
languages = ["zh", "en", "de"]
# Note: The first argument is the script's path
for arg in sys.argv[1:]:
for language in languages:
my_command = ["trans", "-brief", ":"+language, arg]
content = sp.check_output(my_command, env=my_env)
content = content.decode("utf-8")
item = {}
item["title"] = content[:-1]
item["icon"] = language+"_flag.png"
items.append(item)
print(json.dumps(items))
其中：
  • my_env 表示软件路径。
  • languages 表示目标语言，包括 zh（中）、en（英）、de（德）。如果你模仿的动作想要翻译为其它语言，如日语，则在其中填入 jp，其它语言代号还请参阅 translate-shell 官网。
  • check_output 表示运行后面的命令并获取输出的结果。
  • items 表示要在 LaunchBar 上显示的条目，其每一个条目的内容用 title 表示，每一个条目的图标用 icon 表示，图标可以是资源文件中的图片文件（用文件路径表示），也可以是 Font Awesome Icon（用 Icon Identifier 表示）。

另外还需要在 Action Editor 中 Scripts 标签页中勾选如下项目：

  • Requires argument：选择此动作按回车后可以输入文字；
  • Accepts string argument：此动作接受字符串；
  • Returns result → Item：此动作会显示结果。
