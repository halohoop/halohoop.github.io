针对第四章《文本操作》，本周共收到以下几个重要的提问。
@Muchun 提问说：
当系统语言为中文时，无法通过英语单词找到 Emoji 表情。请问有什么合适的解决方案吗？
这其实是一个非常重要的问题，可能也有相当一部分的中文系统用户对此感到困扰。由于我使用的系统不是中文，我在写这方面内容的时候没有考虑到这个问题。现在正好借这个机会阐释一下。
LaunchBar 最开始只支持两种语言：英语和德语（因为 LaunchBar 所属的 Objective Development Software GmbH 是一家奥地利公司，而奥地利的官方语言为德语）。后来 LaunchBar 加入了更多语言界面的时候，只是直接粗暴地把仅仅在系统中出现的分类名称（如「应用程序」）以及项目名称（如各项 Emoji，系统 Services）显示为系统语言。
如西班牙语界面下的 Emoji 列表：

西班牙语界面下的系统 Services 列表：

而除了系统中存在对应翻译的项目以外，LaunchBar 大部分功能仍然使用英语进行表示。如下面的界面也是西班牙语界面下的显示内容，但由于都是 LaunchBar 自身的动作，亦显示为英语：

除了西语和英语的不统一造成的混乱，这个问题对西语用户的影响并不算太大，因为西语也是通过输入拉丁字母进行检索的，也就是说，所有的项目都可以像在英语界面上一样直接检索得到。
而这样的做法对中文界面用户并非如此。如中文界面下的 Emoji 列表：

中文界面下的 Services 列表：

当这些项目全部由中文表示的时候，由于在 LaunchBar 中无法输入中文，这些项目都无法直接通过缩写检索出来。
也就是说，中文界面的 LaunchBar 基本上是不可用的，这无疑是 LaunchBar 设计上的一个败笔。
这可能是因为 LaunchBar 的设计团队中并没有中文使用者，无法意识到这个致命的问题，而在增加支持语言时设计人员又抱着一种「Besser als nichts」（聊胜于无）的心态，同时由于中文用户的基数较少，对此也未提出反馈，导致这个问题一直存在至今。
如果你使用的是中文系统，直接打开 LaunchBar 时它会自动以中文作为界面语言进行启动。我们要避免这个过程，而选择英文作为其启动语言。
方法非常简单：打开终端 Terminal 或 iTerm，在其中输入如下命令，再重新启动 LaunchBar 即可：
defaults write at.obdev.LaunchBar AppleLanguages -array en
当然，更习惯使用 LaunchBar 的你可以直接把这行命令直接快速发送到 LaunchBar 后再发送给 Run Terminal Command 或 Run iTerm Command 动作直接运行：
重新启动后的 LaunchBar 中的所有项目就都是以英文显示了。
@tardis_tao 提问说：
  1. LaunchBar 中找不到服务 Convert Text to Traditional Chinese 和 Emoji 表情「😂」。
  2. LaunchBar 中输入 KISS 不显示「😘」。需要先检索到 Emoji，再在列出的结果中进行二级检索，但查找表情的时候只能输入一个字母（如想输入 KISS 时只能输入一个K。
  1. 当然首先要确认你的 LaunchBar 是英文界面。如果不是，请按上面的方法把 LaunchBar 的启动语言在设为英文。
    如你的 LaunchBar 使用了英文界面而仍有这个问题，请在 LaunchBar Index 中在 相应的列表中检查服务 Convert Text to Traditional Chinese 和 Emoji 表 情「😂」是否被激活。
    同时还要确认 Services 和 Emoji 两项的 Options 标签页中的 Access items via sub-search only（仅在二级检索时访问）未被勾选。

  2. 在检索时如果发生只能输入一个字母的情况，请确认输入法是否被切换到了中文。
@Minja 提问说：
读完更加喜欢用 Snippet 了，想继续研究一下。
我在 LaunchBar 里实现过这样的「文本模板」功能，例如：输入「客户名」、「商品名」，它们会自动填充进「『客户名』您好，您订购的『商品名』刚刚出锅，正在加紧送至府上」。
请教一下，这个需求通过 Snippet 能做到吗，我现有的方法过于复杂，每次做新的模板都要改很久。
很遗憾 LaunchBar 的 Snippet 只能添加一个光标占位符，你谈到的这个需求无法通过 Snippet 来实现。但是可以写一个使用 Python 脚本的 LaunchBar 动作来实现它，比如：获取输入的文字（格式为「客户名，商品名」），提取这两个字符串，插入到全文中，再把这个长字符串复制到剪贴板。用这个方法还可以批量把多个长字符串依次写入剪贴板，再通过剪贴板工具依次粘贴。每次只修改待插入输入信息的长字符串。这样实现起来并不算复杂。 使用效果如下：

  • Dodolee
    很久以前就尝试过LaunchBar，但各种原因很快就放弃了，今年订阅了这一系列文章后又把LaunchBar捡了起来，按照教程一点点开始使用各种功能，感觉越用越顺手了。
    不能输入中文感觉是最大的障碍，同时我也觉得不应该使用Alfred的方式，个人认为LaunchBar高效的一个重要原因就在于采用了目前这种通过按键快速筛选项目的方法。其实无论是Windows还是macOS，大家在文件管理器里都习惯使用这种方式快速定位文件，LaunchBar只是继承并发扬光光大而已。
    但作为中文用户，系统中不可避免有一些中文项目，比如中文的文件名、1Password的中文登录信息，Safari的中文历史记录和收藏夹，等等。遇到这些如果只能用方向键浏览，那本该提高的效率反而要大打折扣了。几天前为了使用LaunchBar检索系统配置中的项目和绘文字，我甚至直接将系统语言设置成了英语，好在看到这期的答疑，解决了系统语言的问题。
    至于系统中剩余的中文项目，其实只要LaunchBar在做索引的时候，遇到有CJK文字的项目名时能同时索引罗马字的读音，在不改变现有输入方式的情况下完全可以检索中文。虽然我在官网论坛上试着提了这个Feature Request，但看看论坛的日发帖量和开发人员的响应速度，大概等作者增加功能是不现实了。另外因为LaunchBar只是用了一个简单的plist来存储用户自定义的缩写，我在想如果能找到LaunchBar存储的Index，是不是可以通过脚本定期按照Index中的项目读音自动生成缩写，这样也许可以作为一个workaround临时解决一些中文输入的问题。
    关于snippets，我觉得其实可以不用太勉强用LaunchBar。在工具的选择上，我个人比较看重功能的正交性，这也是为什么比起Alfred我更喜欢LaunchBar的一个原因（另一个原因是颜值：）Alfred试图包含太多功能，比如剪切板历史和snippets，这两个功能我目前用Copied和Typinaor（程序snippets用Dash），至于@Minja的需求，用Typinaor和Dash都可以非常简单优雅的实现，这时候我觉得还是让专业的工具来吧。
    P.S. 如果能详细对比一下几个常见的启动器（Spotlight、Alfred、Keyboard Maestro、LaunchBar）可能会对用户选择一款适合自己的工具有些帮助。

      非常感谢你认真的留言。
      各类中文项目的操作在对应所属的章节中都有涉及。对于中文文件的检索，下一章中会谈到，由于本身不支持中文的特性，LaunchBar 无法为搜索中文文件提供一个有效的解决方法，只能推荐使用其它一些辅助方法。
      至于向开发者反馈的途径，直接发邮件或 Twitter 要比在论坛上发帖回复得快得多。
      诚然有很多更专业的工具来更好地满足更专业的需求。Minja 提到的需求，我并非推荐一定要用 LaunchBar 来实现，只是提供一个使用 LaunchBar 来做这件事的思路。
      就像有些读者谈到，如果为了使用 LaunchBar 来使用英语或拼音来命名文件，无异于削足适履。我也非常认同这种观点。每一个工具都有各自擅长的领域，就像微信的主业是聊天，但它也可以用来收藏文件，但它无法替代 Evernote。用适当的工具完成其擅长的任务，才是一种聪明的做法。上一章中我谈到，LaunchBar 提供了一个剪贴板功能，但我也很少会用到它，因为它不如其它专业工具更完善，要表达的就是这个意思。
      虽然对我而言，它的绝大部分功能都非常快捷、实用，完成的方式也非常优雅，但一个工具的功能是否满足需求，终究是因需求而异的。
      当我在写这份 LaunchBar 教程时，还是希望自己能够更全面地展示它的功能。但一个方法适不适合自己，还是要读者自己来判断和取舍。
