第二章 | 快速启动

跟其它启动器一样，LaunchBar 可以快速启动任意你想启动的应用或功能，甚至它能够做得更好更快。通过这一章，你将学会用 LaunchBar 接管你的大部分启动操作，不再需要从 LaunchPad 的图标列表中用鼠标指针选择图标打开应用。

从这个例子可以看到，用 LaunchBar 进行检索，所输入的字符只要满足在目标名称中按照所输入的顺序排列（可以不相临），就可以找到它。所输入的的字符串（Abbreviation），在本教程中译作「缩写」。
LaunchBar 使用了 Adaptive Abbreviation Search Algorithm (AASv4，自适应缩写检索算法) 。借助此算法，LaunchBar 可以学习用户的输入习惯，比如当你输入一个缩写，如果目标检索结果没有出现在结果列表中的最上面，当你用上下方向键选中并回车启动它后，LaunchBar 会记忆你的选择，当你下一次再输入相同的缩写后，LaunchBar 会在检索结果中把目标程序排到上面。也就是说，当你使用 LaunchBar 越多，就会把它训练得越聪明，越了解你的使用习惯，也就会越快找到你想要的结果。

技巧 - 快速打开（Instant Open）

除了按回车打开目标程序，也可以在输入缩写最后一个字符时按住此键，快速打开对应的程序。例如：通过缩写 SAF 快速打开 Safari：
  1. 依次输入 S，A；
  2. 按住 F 直至 Safari 快速启动。

⚙️ 设置 - 重新输入延迟（Retype Delay）

也许在尝试的过程中你发现了，在输入缩写的时候如果发生停顿，则 LaunchBar 会抛掉停顿前所输入的字符，而将停顿后输入的字符识别为重新输入。LaunchBar 的这个设计意义在于：当输入错误之后，只需稍微停顿一下，就可以重新输入，节省了删除错误字符的时间。这个设计在刚上手时可能会让用户不习惯，因为当一边思考检索内容一边输入的时候，很容易停顿，继而被迫重新输入。很多人都抱怨这个设计使 LaunchBar 使用起来不太友好。按下 ⌘ , 打开配置，在 General 标签页下的 Retype Delay（重新输入延迟） 项中可以调节重新输入的停顿间隔：

当你一开始不适应这个设计时，可以把这个间隔适当调得长一点。随着操作的熟练，可以再把它调短，那时你会发现，这其实是个很巧妙的设计。

技巧 - 立即抛掉当前输入内容

想要立即抛掉当前输入的内容重新输入，也可以按下 ESC 或重新按下 ⌘ 空格。

这些系统操作每一项在 LaunchBar 中都一个 Action，在本教程中译作「动作」。动作是 LaunchBar 中最重要的核主功能之一，你在本教程的每一章中都接触到各种不同的动作，它们能在方方面面简化你的操作。

关闭程序

除了快速启动，也可以使用 LaunchBar 快速关闭程序，这个操作在程序卡死的时候非常实用。
例如：如果你的 Safari 卡死了，要用 LaunchBar 关闭它，具体步骤如下：
  1. 在 LaunchBar 在通过缩写如 SAF 选中 Safari；
  2. 按 Tab 键，此时 Safari 名字后面出现了一个 ⇥；
  3. 再通过缩写如 QUIT 找到 Quit Application 或 Force Quit Application 动作，按回车以退出 Safari。
以上的操作，是通过 Tab 键把选中的程序 Safari 发送给 (Force) Quit Application 这个动作。

❗️须知 - 发送

在 LaunchBar，发送 也是一个最常用的核心功能之一，不只可以发送程序，你可以发送 LaunchBar 中几乎一切项目，对这些被发送的项目进行一系列的操作。在接下来的章节中，你会了解发送功能丰富的可能性。
（ 这里再提供一个切换程序界面语言的小技巧，这个方法也是运用了 LaunchBar 的发送功能把程序发送给了另一个程序。）

⌨️ 快捷键 - 指定缩写

如在检索结果名称上点开的菜单所示，也可以用快捷键 ⌥ ⌘ A 为项目指定缩写。

为中文应用指定缩写

对于名称为中文的应用，由于 LaunchBar 不支持中文检索，无法直接检索到这些应用。针对这些应用，就可以为它们分配一个缩写，以便在 LaunchBar 在检索到它。
除了程序，在 LaunchBar 可以给任意项目指定缩写，在之后的章节中涉及到的所有检索项目你都可以尝试为它们指定缩写。
可在分配缩写的时候需要先在 LaunchBar 中找到这个项目，不借助缩写如果在 LaunchBar 中找到它呢？还有以下两种方法：
  • 在 LaunchBar 中输入缩写如 APP 找到 Applications 分类，按 → 方向键打开这个分类并按上下方向键查看，找到这个软件为其指定缩写。
  • 在 Finder 中进入 Applications 文件夹，选中这个软件后长按 ⌘ 空格，直到唤出 LaunchBar 界面，此时这个软件显示在 LaunchBar 界面中，此时也可以为其指定缩写。
这两种方法分别使用了 LaunchBar 的另外两项重要功能：二级检索和快速发送。本章后面的会详细介绍二级检索功能，下一章《网络操作》中会对快速发送功能进行介绍。

初识 LaunchBar Index

LaunchBar 当然不只可以用于启动程序和一些系统命令，还可以检索各种不同的项目。在 LaunchBar 中所有的可检索的项目都被整理在一个索引（Index）中。
在唤出 LaunchBar 后把鼠标移到界面的最右边，此时这里会显示一个齿轮图标，点击它，可以弹出另一个菜单（LaunchBar Menu），本教程中将其译作「LaunchBar 菜单」。在这个菜单中选择 Index → Show Index，可以打开 LaunchBar Index 窗口：
或在唤出 LaunchBar 后，按下 ⌥ ⌘ I，也可以打开 LaunchBar Index 窗口。
可以看到，检索程序只是 LaunchBar 的极其丰富的功能中的一项，这里面所列出的其它所有内容，都会在之后的内容中进行介绍。

在上图中，每一个项目的最右边有一个勾选框，表示是否在 LaunchBar 中检索这个项目。例如列表中前两个程序 AdobeApplicationManager 和 AdobeIPCBroker，平时并不会用到，就可以选中该项目时按空格键，把选框中的勾去掉，在 LaunchBar 中检索时这两项就不会列出在检索结果中了；再比如之前所说到的，我用 Spark 替换了 Mail， 我不希望输入 MAIL 缩写的时候检索结果中出现 Mail，就可以在这里把 Mail 取消勾选。
