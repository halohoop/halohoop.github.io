第五章 | 文件检索与操作

| 本文为付费栏目文章，您已订阅，可阅读全文 |
LaunchBar 提供了非常强大的文件检索和处理功能。除了迅速定位文件以外，还可以轻松实现对文件的复制、移动、重命名等基础操作，也可以同样轻松地对其进行更复杂的压缩、转换等不同处理。

把文件传递至 LaunchBar 的方法

和文本操作类似，在 LaunchBar 中处理文件可以通过快速发送把文件传递至 LaunchBar 再进行操作，也可以先通过检索找到目标文件再对其进行操作。

通过快速发送传递至 LaunchBar

与快速发送文本的方式一样，在 Finder 中选择好文件之后按下快速发送快捷键或长按 ⌘Command-空格 把它们传递到 LaunchBar。可以快速发送一个文件，也可以传送多个文件。
同时也可以把一串路径选中快速发送至 LaunchBar。

文件检索

LaunchBar 中也可以直接在设定的范围内检索文件。

⚙️ 设置 - 定义检索范围

首先要在 LaunchBar Index 中选择检索范围，即 Launchbar 监控的所有文件夹。在激活的 LaunchBar 后按下 ⌥Option-⌘Command-I，打开 LaunchBar Index，点击左上角的 New 对应的加号，在弹出的菜单中选择 Add Folder…，在其中添加检索文件的范围：
添加的文件夹会在 LaunchBar Index 左侧栏的下面的 Files & Folders 中列出来：

如上图所示，设置检索范围时，我建议只添加用得到的文件夹，如：桌面、文档、下载、Dropbox 等。当然可以选择把整张硬盘全都放在检索范围中，但这样检索结果中就会列出很多系统文件等不希望出现的文件，并且会大大增加 LaunchBar 更新索引的时间，造成卡顿，所以我并不建议这样做。
同样的，在这里列出的文件夹及其内部的单独项目，都可以禁用掉，以避免它在全局检索时出现在检索结果中。

尝试检索

添加完成之后，就可以检索这些范围内的所有文件了。检索方法跟前面的介绍的一样，输入目标文件的任意缩写即可。例如：要检索一个名为 Lebenslauf.pdf 的文件， 可以输入缩写 LEBN，就可以找到它了：

⚙️ 设置 - 设置检索范围细节

在 LaunchBar Index 中选中一个加入的文件夹，点击右上角的 Options 按钮，则可以对这个文件夹的检索进行更细节地设置：

  • Access items via sub-search only：仅允许二级检索时访问项目，例如，对 Dropbox 勾选此项，则在全局检索时无法检索出其中的文件，需先检索出 Dropbox，在它上面按 回车或 → 方向键，才可以对其中的内容进行二级检索；
  • Search Scope：设置文件夹检索的深度，可以选择检索时不检索此文件夹的下一级子文件夹中的内容，仅检索此文件夹中以下第一级至第七级的子文件夹中的内容，或检索全部子文件夹；
  • Search for：设置检索文件的类型；
  • Skip Subfolders：设置不检索的子文件夹，可以按提示把不需要检索的子文件夹拖入列表，或直接输入不需要检索的子文件夹的名字，可以使用通配符 ? 或 *；
  • Exclude Items：设置不检索的文件，方法依提示，如上；
  • Exclude Aliases：排除快捷方式；
  • Search file package contents：检索文件包的内容。

文件的基础操作

在 LaunchBar 中对文件的操作也基本与在 Finder 中的操作方式相同：
  • 快速查看（Quick-Look）：空格或 ⌘Command-Y；
  • 删除文件：⌘Command-退格；
  • 查看信息：⌘Command-I；
  • 复制文件：⌘Command-C；
  • 新建文件夹：⇧Shift-⌘Command-N；
  • ……
也有一些不同于在 Finder 中的操作：
  • 列出所有子文件夹及子文件： ⇧Shift-空格（如在章节《网络操作 | ❗️ 须知 - 查看所有各级子分组和子项目》中所述）；
  • 查看文件夹中的隐藏文件：⇧Shift-⌥Option-→；
  • 在 Finder 中显示：⌘Command-回车；
  • 重命名文件：⇧Shift-⌘Command-R；
  • 在文件夹上或文件所在路径上打开 Terminal：⌘Command-T；
  • 在 LaunchBar Index 中显示此项目：在选中文件上按 ⇧Shift-⌘Command-I，这个快捷键可用于除文本以外的任意项目，不限于文件；
  • ……
这些功能都可以在选中项上点击鼠标左键打开动作菜单（或按快捷键 ⌃Control-⌘Command-→）进行查看。这些功能不需要全部记住，自己最常用的几项会在使用过程中形成肌肉记忆，不常用的操作直接动作菜单中选择即可。

复制、移动和建立链接

利用 LaunchBar 检索得到文件后可以紧接着对其进行操作，例如复制、移动等动作。
例如：我想要把我的文件 Lebenslauf.pdf 移到到 Dropbox 文件夹中，步骤如下：
  1. 通过缩写检索到文件 lebenslauf.pdf；
  2. 选择此文件后，按 ⇥Tab 对其进行发送；
  3. 输入缩写 DROP，找到 Dropbox 文件夹；
  4. 按下回车，得到如下界面：

    用上下方向键或数字键选择 Move to “Dropbox”，就可以文件移动到 Dropbox 文件夹中了。
如图所示，除了移动，把文件/文件夹发送给文件夹的操作还包括以下几种可能性：
  • Copy to：复制到；
  • Make Alias：创建别名；
  • Make Hard Link：创建硬链接；
  • Make Symbolic Link (Absolute)：创建软链接（绝对）；
  • Make Symbolic Link (Relative)：创建软链接（相对）。
另外，除了在 LaunchBar 中进行移动操作以外，也可以通过 Drag & Drop 的方式进行文件的移动。例如：想要把桌面上所有的 PDF 文件移动到一个 Finder 已经打开的文件夹中。操作步骤为：在 LaunchBar 中选中目标 PDF 文件，用鼠标将其拖入 Finder 窗口中。
其实，LaunchBar 对文本同样支持 Drag & Drop 操作。

选择打开方式

与选择浏览器打开网址的操作类似，也可以选择不同的应用来打开文件。
例如：在我的电脑上默认的 PDF 阅读器是 PDF Expert，当我想把一个文件用 Preview 打开时，操作步骤如下： 1. 通过快速发送或检索找到要打开的 PDF 文件； 2. 按 ⇥Tab 键，输入缩写 PRE，找到 Preview，按回车打开。

对文件执行动作

与对文本执行动作类似，LaunchBar 可以一样方便得对文件执行各样动作。下面列举是一些常用的例子。

压缩文件

在 Finder 中选择要压缩的所有文件，快速发送至 LaunchBar，再通过缩写如 ZIP 找到动作 Compress (ZIP Archive)，回车即可创建压缩文件。

添加标记

同样，用快速发送或检索的方法使 LaunchBar 获取文件，输入 缩写如 TAG 找到动作 Add Tag，回车即可给文件添加标记。
通过 Remove Tag 则可以取消文件的标记。

更多

请再自己尝试一下以下的动作：
  • Send via AirDrop
  • Evernote – New Note With File
  • Flip Vertical（对图像使用）
在 LaunchBar 中也可以自制对文件的动作，详见章节《动作》。

查看文件信息

如在章节《快速启动 | 二级检索 | ❗️ 需知 - 查看项目信息》中所述，在文件上按 → 可以查看其详细信息，如名称、路径、创建和修改时间、版本、所处类别等等。

❗️ 须知 - 多项选择

如前面所述，当然也可以通过检索的方式找到文件。但之前讲到的，都是在 LaunchBar 中检索得到一个项目，如一个程序、一个文本、一个文件。其实除了把多个文件进行快速发送以外，LaunchBar 也支持在检索时进行多项选择，以对其进行同时操作。有以下两种方法：

选择同一个结果列表下的文件

如果要选择一个文件夹中的所有文件，操作方法和在 Finder 中一样，直接按 ⌘Command-A 即可。这时 LaunchBar 界面的第一行会显示所选的文件的数量：
如果要选择一个文件夹中列出的连续的文件，也和在 Finder 中类似，按住 ⇧Shift 键再按上下方向键即可：
你也许发现了，用这两种方法选择多个文件，文件其实并不一定要处于同一个路径下，只要它们出现在同一个检索结果中，就可以这样选择它们。例如：想要选择 LaunchBar 监控的所有文件夹中的所有 PDF 文件，操作步骤如下：
  1. 通过缩写 PDFD 找到 PDF Documents 分类；
  2. 按下回车或向右方向键，进入此分类，再按下 ⌘Command-A 即可选中所有的 PDF 文件。

选择分散在各处的文件

选择多个项目的操作，在 LaunchBar 中称为 Staging （工作台），寓意非常形象，即把要处理的项目先堆放在工作台上，现进行下一步的操作。
除了上一小节介绍的两种特殊情况下的两种方法以外，更普适的操作是：在一个文件上按逗号 , 键，就把它堆在了工作台上，即可再进行下一个文件的选择。
例如：我想要找到电脑中所有名称的含有 Lebenslauf 的文档以及桌面上的 Anschreiben.pdf 文件压缩成 ZIP 文件，操作步骤为：
  1. 输入缩写 LEBENSLAUF 找到所有名称中含有 Lebenslauf 的文档，按 , 依次选中；
  2. 此时高亮色仍为蓝色，表示全局搜索，输入缩写进入桌面（Desktop），高亮色变为橙色，表示进入二级搜索状态，输入缩写检索 Anschreiben.pdf，按 , 选中。
这样即可把目标文件全都选出来，以待进行下一步操作，如压缩。
如果把顺序反过来，先选择桌面上的 Anschreiben.pdf 再选择电脑中所有名称的含有 Lebenslauf 的 PDF 文档，操作步骤为：
  1. 输入缩写进入桌面（Desktop），高亮色变为橙色，表示进入二级搜索状态，输入缩写检索 Anschreiben.pdf，按 , 选中；
  2. 此时我们需要重新进入全局搜索状态，重新按下 ⌘Command-空格，高亮色变为蓝色，表示进入了全局搜索状态，此时再输入缩写 LEBENSLAUFPDF 找到所有含有 Lebenslauf 的 PDF 文档，按 , 依次选中。
多项选择不仅适用于文件，它和其它所有操作一样，适用于 LaunchBar 中的所有项目，比如，你可以在 Safari 的书签中打开所有用某缩写筛选出来的网页。

检索中文文件

由于 LaunchBar 不支持中文的检索，前面的章节中介绍了对程序、Snippet 等项目可以对其进行指定缩写。对于名称为中文的文件，由于此类文件数量可能非常大，也来分别指定缩写是不现实的。
除了更多地为文件用字母命名以外，我建议以下两种方法来克服这个问题：
  1. 对重要、常用文件进行标注。那么检索这些文件时，可以通过缩写找到它所处的标注，再在此标注的列表中用上下方向键进行查看。
  2. 使用 Search in Spotlight 动作。使用 Spotlight 不但可以通过名称搜索文件，也可以通过日期、内容搜索文件，对中文也有非常好的支持。故搜索中文文件的时候，Spotlight 是一个很好的工具。
Spotlight 还支持使用数据属性对搜索结果进行限制，以下是如何在搜索中使用数据属性的示例（部分例子取自 Apple Support 页面）：
  • name: 简历：搜索名称含关键词「简历」的文件；
  • 旅行 kind:document：仅在文稿中搜索关键词「旅行」；
  • author: 小明： 搜索作者为小明的所有项目；
  • 会议 date:tomorrow： 搜索计划明天召开的会议；
  • kind:images created:5/16/16： 搜索特定日期创建的图像；
  • modified:<=6/29/16： 搜索特定日期或特定日期前修改的项目。
以上就是关于文件检索与操作的全部内容了。
可以看到，对文件的检索和操作与之前几章所述的对其它项目的检索与操作方法都是一致的，如二级检索、快速发送、发送等方法，在对于文件、文本等各类项目全都拥有统一的操作逻辑，这正是 LaunchBar 的魅力的一个表现。
