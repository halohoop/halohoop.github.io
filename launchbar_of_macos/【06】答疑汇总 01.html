本周是这份教程上线的第一周，只收到一位读者的提问：
@Alanku 在《第二章 | 快速启动》的评论区说：
当我找档案时（Keynote File），显示的档案数不多。直至我测试先打 KEY，然后在 Keynote 按右，再查 File Name，才可以出现较多的档案。找档案需求简单，但却是日常最重要及基本的功能，谢谢分享有关技巧。
这个问题放在第二章有点超纲，涉及到文件检索的问题，会在《第五章 | 文件操作》中提到。
你用到了两种检索 Keynote Presentations 的方法。第二种「在 Keynote 按 →方向键，再查文件名」是本章介绍的方法，在应用程序上按 → 方向键，查看这个应用程序最近访问的文件。
第一种「在 Keynote Presentations 按 →方向键，再查文件名」，是在 Keynote Presentations 这个文件类别中进行二级检索，查看 Index 中所有此类别的文件。
你谈到用这个方法显示的文件数不多，找不到想要搜索的文件，是因为这个文件没有被纳入到 Index 中，也就是说，你需这个文件所处的路径加入到 LaunchBar 的监视范围以内。
举例说明：我在桌面上建立了一个 Keynote 文件 keynote on desktop.key：

当桌面未被纳入 LaunchBar Index 时，此文件就不会在 Keynote Presentations 类别中显示：
当桌面被纳入 LaunchBar Index 时，此文件才会在此类别中显示：
要把某个文件夹如纳入 LaunchBar Index，具体方法是：
  1. 唤出 LaunchBar 后按 ⌥ ⌘ I 打开 LaunchBar Index；
  2. 选择 LaunchBar Index 窗口左上角的 + 号按钮，在弹出的菜单中选择第一项 Add Folder…，再在其中选择文件夹并加入及可。
这样，加入的文件夹内的所有文件（包括各级子文件夹及其内部所有文件）就都被纳入 LaunchBar 的检索范围了。
现在你一定急着想直接把根目录加入到 LaunchBar Index 中。在第五章中会谈到，这并不是一个好的做法。因为这样会在检索结果中加入非常多你根本不想要搜索的系统文件、配置文件等等，另外也会造成 LaunchBar 实时刷新 Index 时所消耗的时间。因此，设置检索范围时，我建议只添加用得到的文件夹，如：桌面、文档、下载、Drobpox 等。
